# 📖 Introduction

One of the most common runtime errors in Java is

```text
NullPointerException
```

Java 8 introduced the **Optional** class to reduce null-related bugs and make code more readable.

Instead of returning

```java
null
```

a method can return

```java
Optional<T>
```

making it explicit that a value may or may not be present.

---

# ❓ What is Optional?

`Optional` is a container object that may contain a value or may be empty.

```java
Optional<String> name;
```

The value can be

- Present
- Empty

This eliminates many manual null checks.

---

# 🤔 Why Optional?

Without Optional

```java
if(name != null){

    System.out.println(name);

}
```

With Optional

```java
Optional<String> name = Optional.of("Java");

System.out.println(name.get());
```

The code becomes cleaner and easier to understand.

---

# 🏗️ Creating Optional Objects

## of()

Used when the value is guaranteed to be non-null.

```java
Optional<String> name = Optional.of("Java");
```

---

## ofNullable()

Accepts both null and non-null values.

```java
Optional<String> name = Optional.ofNullable(value);
```

---

## empty()

Creates an empty Optional.

```java
Optional<String> name = Optional.empty();
```

---

# ⚙️ Important Methods

## isPresent()

Checks whether a value exists.

```java
if(optional.isPresent()){

}
```

---

## get()

Returns the stored value.

```java
optional.get();
```

⚠️ Never call `get()` without checking first.

---

## orElse()

Returns a default value.

```java
String name = optional.orElse("Guest");
```

---

## orElseGet()

Returns a value generated by a supplier.

```java
optional.orElseGet(() -> "Default");
```

---

## orElseThrow()

Throws an exception if the value is absent.

```java
optional.orElseThrow();
```

---

## ifPresent()

Executes code only if a value exists.

```java
optional.ifPresent(System.out::println);
```

---

## map()

Transforms the stored value.

```java
optional.map(String::toUpperCase);
```

---

## filter()

Returns the value only if it satisfies a condition.

```java
optional.filter(name -> name.length() > 5);
```

# 🔄 Optional Workflow

```text
Value

↓

Optional.ofNullable()

↓

Value Present?

↓

Yes

↓

Use Value

↓

No

↓

Return Default

↓

Continue Execution
```

# ✅ Best Practices

- Prefer `ofNullable()` when values may be null.
- Use `orElse()` for default values.
- Avoid calling `get()` without checking.
- Use `ifPresent()` for conditional execution.
- Use `map()` for transformations.
- Keep Optional usage limited to return types.

---

# ❌ Common Mistakes

- Calling `get()` directly.
- Using Optional for every variable.
- Wrapping primitive values unnecessarily.
- Ignoring `orElse()` or `orElseThrow()`.
- Returning null instead of `Optional.empty()`.
